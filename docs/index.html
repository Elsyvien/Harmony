Harmony Application Documentation

Harmony is a Discord-style chat platform built with Fastify + Prisma (backend) and React + Vite (frontend). This page documents system design, REST and WebSocket contracts, and function-level APIs from both backend and frontend modules.

REST API WebSocket Events Backend Functions Frontend Functions

Overview

Core Features

JWT auth, channel messaging, uploads, friendships, direct messages, voice signaling, admin controls, role-based permissions, and live events over WebSocket.

Tech Stack

Backend: Fastify, Prisma, Zod, JWT, multipart uploads.
Frontend: React 19, React Router, TypeScript, Vite.
Tests: Vitest backend service tests.

Entry points: backend/src/server.ts, backend/src/app.ts, web/src/main.tsx, web/src/App.tsx.

Architecture

Layer Primary Modules Responsibility
HTTP & WS Transport backend/src/routes/*, backend/src/plugins/ws.plugin.ts Validates input, authenticates, delegates to services, emits live events.
Domain Services backend/src/services/* Business rules for auth, messages, channels, friendships, and admin actions.
Data Access backend/src/repositories/* Typed persistence layer over Prisma models.
Frontend API + State web/src/api/*, web/src/store/auth-store.tsx, web/src/hooks/* Client requests, auth/session persistence, socket lifecycle, preferences.

Data Model

Defined in backend/prisma/schema.prisma.

Model Key Fields Notes
User username, email, role, suspension fields Roles: OWNER|ADMIN|MODERATOR|MEMBER.
Channel name, type, dmKey, voiceBitrateKbps Types: PUBLIC|VOICE|DIRECT.
Message content, attachment fields, FK to channel/user Attachment metadata stored inline on message row.
Friendship userAId, userBId, requestedById, status Status: PENDING|ACCEPTED.
AppSettings allowRegistrations, readOnlyMode, slowModeSeconds Global runtime controls used by auth/message services.

HTTP API: Auth

Base URL: http://localhost:4000. Auth header: Authorization: Bearer <jwt>.

Endpoint Input Response
POST /auth/register { username, email, password } with zod validation { token, user } (201)
POST /auth/login { email, password } { token, user }
POST /auth/logout Bearer token required No content (204)
GET /me Bearer token required { user }
GET /health none { ok: true }

HTTP API: Channels, Messages, Uploads

Endpoint Auth / Role Purpose
GET /channels User List accessible public/voice/direct channels for current user.
POST /channels Admin Create TEXT or VOICE channel.
DELETE /channels/:id Admin Delete non-direct channels. Cannot delete global or last public channel.
PATCH /channels/:id/voice-settings Admin Update voiceBitrateKbps (16..1536), broadcasts channel:updated.
POST /channels/direct/:userId User Open/create DM channel (requires accepted friendship).
GET /channels/:id/messages User List messages with optional before and limit.
POST /channels/:id/messages User Create message with optional attachment metadata, then broadcast.
POST /uploads User Upload a single file (max 8MB), returns attachment payload.
Message rules are enforced in MessageService.createMessage: read-only mode, slow mode cooldown, max content length, and non-empty content/attachment requirement.

HTTP API: Friends

Endpoint Purpose
GET /friendsList accepted friendships.
GET /friends/requestsList incoming/outgoing pending requests.
POST /friends/requestsSend friend request by username.
POST /friends/requests/:id/acceptAccept incoming request.
POST /friends/requests/:id/declineDecline incoming request.
POST /friends/requests/:id/cancelCancel outgoing request.
DELETE /friends/:idRemove accepted friend relationship.

HTTP API: Admin

Endpoint Role Purpose
GET /admin/stats Admin Runtime metrics for node/system/database counts.
GET /admin/settings Admin Read global app settings.
PUT /admin/settings Admin Update registration flag, read-only mode, and slow-mode seconds.
GET /admin/users Owner/Admin List users for moderation/role management.
PATCH /admin/users/:id Owner/Admin Update target role with hierarchy checks.
DELETE /admin/users/:id Owner/Admin Delete target account with hierarchy checks.

HTTP Errors

Error shape: { code: string, message: string }.

Code (Examples) When Raised
UNAUTHORIZED, INVALID_SESSIONInvalid/expired token or deleted user session.
VALIDATION_ERRORZod payload/param/query validation failure.
FORBIDDENRole or ownership restriction.
ACCOUNT_SUSPENDEDUser is currently suspended.
READ_ONLY_MODE, SLOW_MODE_ACTIVEAdmin runtime chat restrictions for non-admin users.
CHANNEL_NOT_FOUND, USER_NOT_FOUNDMissing resources.
INTERNAL_ERRORUnhandled server exception.

WebSocket Protocol

Endpoint: ws://localhost:4000/ws. Envelope: { type, payload }.

Client → Server Events

auth, channel:join, channel:leave, voice:join, voice:leave, voice:signal, message:send.

Server → Client Events

auth:ok, channel:joined, channel:left, message:new, dm:new, channel:updated, presence:update, voice:state, voice:signal, friend:request:new, friend:request:updated, error.

Event Payload Notes
auth{ token }Must be first event; server replies auth:ok.
channel:join{ channelId }Validates access, then subscribes socket.
voice:join{ channelId }Only allowed for voice channels visible to caller.
voice:signal{ channelId, targetUserId, data }Relays WebRTC signaling to specific participant.
message:send{ channelId, content }Creates and broadcasts message (same rules as REST).

Backend Function & Class API

This section documents major exported APIs from backend modules and key internal helper functions that affect behavior.

Module Public API Behavior
backend/src/app.ts buildApp() Composes plugins/routes/services, ensures default channel, sets error handler.
backend/src/config/env.ts loadEnv(), type Env Parses and validates environment variables with zod.
backend/src/services/auth.service.ts AuthService.register/login/getById Handles credential checks, password hashing, user bootstrap role logic.
backend/src/services/channel.service.ts ensureDefaultChannel, listChannels, createChannel, deleteChannel, updateVoiceChannelBitrate, ensureChannelExists, ensureChannelAccess, getChannelSummaryForUser, openDirectChannel Channel lifecycle and DM creation with friendship enforcement.
backend/src/services/message.service.ts listMessages, createMessage Enforces slow mode, read-only mode, length, and empty-message constraints.
backend/src/services/friend.service.ts listFriends, listRequests, sendRequest, acceptRequest, declineRequest, cancelRequest, removeFriend Friend request state machine and participant permission checks.
backend/src/services/admin.service.ts getServerStats Returns process/system/database stats for dashboard.
backend/src/services/admin-settings.service.ts getSettings, updateSettings, getSlowModeRetrySeconds, markMessageSent Persists runtime switches and in-memory slow-mode timestamps.
backend/src/services/admin-user.service.ts listUsers, updateUser, deleteUser Role hierarchy enforcement for user management.
backend/src/plugins/ws.plugin.ts wsPlugin, decorated gateway broadcastMessage, notifyUsers, broadcastSystem Socket auth, subscriptions, presence, DM notifications, voice state/signaling.
backend/src/repositories/*.ts PrismaUserRepository, PrismaChannelRepository, PrismaFriendshipRepository, PrismaMessageRepository Data access APIs consumed by services.
backend/src/utils/*.ts AppError, isAdminRole, isPrivilegedRole, isSuspensionActive Error transport and authorization/suspension helpers.
